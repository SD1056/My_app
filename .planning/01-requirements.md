# 요구사항

## 핵심 사용자 시나리오

### 시나리오 1 — 지출 자동 기록 (가장 빈번한 케이스)

**페르소나:** 직장인 지수 (28세), 매달 가계부를 쓰려다 3일 만에 포기해온 경험 있음

**상황:** 편의점에서 커피를 사고 앱을 켠다.

**흐름:**
1. 앱을 열면 "오늘 오전 9시, 편의점 커피 4,500원 — 맞나요?" 라고 AI가 먼저 제안
2. 지수는 "확인" 한 번만 탭
3. 기록 완료. 이번 달 커피 지출 누적 표시

**성공 조건:** 제안이 맞아서 수정 없이 수락, 5초 이내 완료

---

### 시나리오 2 — 루틴 학습 및 자동완성 (AI 핵심 가치)

**페르소나:** 운동을 시작한 민준 (32세), 매일 같은 운동을 하지만 기록을 자주 빠뜨림

**상황:** 저녁 7시에 헬스장 갔다가 돌아온 뒤 앱을 켠다.

**흐름:**
1. 앱이 "오늘 헬스장 운동 (어제와 동일: 스쿼트 3세트, 벤치프레스 3세트) 기록할까요?" 제안
2. 민준이 세트 수 하나만 수정하고 저장
3. 앱이 "이번 주 4회 달성!" 뱃지 표시

**성공 조건:** 3주 사용 후 제안 정확도가 체감될 만큼 높아짐

---

### 시나리오 3 — 신규 패턴 입력 (온보딩 & 첫 사용)

**페르소나:** 앱을 처음 설치한 혜린 (25세), AI 제안이 아직 없는 상태

**상황:** 점심 식사 후 처음으로 식단을 기록하려 한다.

**흐름:**
1. "+" 버튼 탭 → 카테고리(식단) 선택
2. 음식 이름 타이핑 시작 → 자주 먹는 음식 자동완성 목록 표시
3. 분량 입력 → 저장
4. 앱: "비슷한 시간대에 또 기록하면 다음번엔 자동으로 제안해드릴게요"

**성공 조건:** 첫 입력이 30초 이내, 앱이 학습 중임을 사용자가 인지

## 기능 분류 (MoSCoW)

### Must Have — 없으면 제품이 아님

| 기능 | 설명 |
|------|------|
| 수동 데이터 입력 | 카테고리 선택 → 값 입력 → 저장 |
| AI 자동 제안 | 과거 패턴 기반으로 다음 입력 내용 선제안 |
| 제안 수락 / 수정 / 거절 | 원탭 수락, 인라인 수정, 거절(학습 반영) |
| 기록 목록 & 히스토리 | 날짜별 입력 기록 조회 |
| 로컬 데이터 저장 | 오프라인에서도 기록 가능 |

### Should Have — 있어야 완성도가 높음

| 기능 | 설명 |
|------|------|
| 카테고리 커스터마이징 | 사용자가 자신만의 기록 유형 추가 |
| 통계 / 요약 뷰 | 주간·월간 집계 차트 |
| 제안 정확도 피드백 | 거절 시 이유 선택 → 모델 개선 |
| 푸시 알림 | 기록 안 한 경우 부드러운 리마인더 |

### Could Have — 있으면 좋지만 후순위

| 기능 | 설명 |
|------|------|
| 위젯 (홈화면) | 앱 열지 않고 빠른 수락 |
| 내보내기 (CSV / PDF) | 월별 리포트 다운로드 |
| 다크 모드 | UI 테마 선택 |
| 음성 입력 | 말로 기록 |

### Won't Have (v1) — 이번 버전에서 명시적으로 제외

| 기능 | 이유 |
|------|------|
| 외부 앱 자동 연동 (카드사, 건강앱) | 인증·API 복잡도, v2로 이연 |
| 팀/공유 기능 | B2C MVP 집중, 복잡도 증가 |
| 웹 버전 | 모바일 우선 전략 |
| 정기 결제 / 구독 관리 | 수익화는 PMF 확인 후 |
