# ADR-0006: AI 제안을 온디맨드(+ 버튼 탭 시)로 호출

- **상태**: Accepted
- **결정일**: 2026-05-18

---

## 맥락

AI 제안을 언제 생성할지 결정해야 했다.
크게 두 가지 전략이 있다:
- **백그라운드 프리페치**: 앱 실행 시 또는 주기적으로 미리 제안 생성
- **온디맨드**: 사용자가 + 버튼을 탭할 때만 API 호출

## 결정

사용자가 **+ 버튼을 탭할 때** Claude API를 호출한다.
`FutureProvider.autoDispose`를 사용해 시트가 닫히면 요청이 자동 정리된다.

## 근거

- **배터리**: 백그라운드 주기 실행은 배터리를 소모하고, 실제 사용 안 할 때도 API를 호출
- **비용**: 온디맨드는 사용자가 실제로 기록하려 할 때만 API 비용 발생
- **최신성**: 제안 시점에 최신 기록 패턴을 반영한 결과 제공
- **단순성**: 백그라운드 워커, 캐시 만료 로직 불필요

## UX 처리

API 응답 대기 중에는 로딩 스피너를 바텀 시트에 표시한다.
응답이 느릴 경우(10초 타임아웃) 수동 입력 폼으로 자동 전환한다.

## 대안

| 대안 | 제외 이유 |
|------|-----------|
| 앱 실행 시 프리페치 | 매번 앱 실행마다 API 호출, 사용 안 하면 낭비 |
| 백그라운드 주기 갱신 (WorkManager) | 플랫폼별 설정 복잡, 배터리 소모 |
| 제안 결과 30분 캐싱 | 캐시 무효화 로직 추가 복잡도, MVP에 불필요 |

## 결과

**긍정적:**
- API 비용이 실제 사용량에 정비례
- 배터리·네트워크 사용 최소화
- 구현 복잡도 낮음

**부정적:**
- + 버튼 탭 후 짧은 로딩 대기 발생 (보통 1~3초)
- 네트워크가 느린 환경에서 UX 저하 가능

## 향후 고려

DAU가 증가하고 제안 정확도가 검증되면,
자주 사용하는 시간대에 한해 선택적 프리페치를 도입할 수 있다.
