# ADR-0003: 온디바이스 ML 대신 Claude API 사용

- **상태**: Accepted
- **결정일**: 2026-05-18

---

## 맥락

AI 패턴 학습 및 자동 제안 기능을 구현하는 방법을 결정해야 했다.
기기 내에서 처리할지, 클라우드 AI를 호출할지 선택이 필요했다.

## 결정

**Anthropic Claude API** (`claude-sonnet-4-6`)를 온디맨드로 호출한다.
기기 내에서는 패턴 빈도 분석만 수행하고, 실제 제안 생성은 Claude에 위임한다.

## 근거

- 온디바이스 ML 모델을 직접 학습·배포하는 비용과 복잡도가 MVP 수준을 초과
- Claude는 자연어 이해 능력이 뛰어나 패턴 요약 → 제안 생성에 적합
- 패턴 요약 텍스트(개인정보 제거)만 전송하므로 개인정보 노출 최소화
- API 호출 당 비용은 제안 횟수 제한(일 10회 권장)으로 관리 가능

## 데이터 전송 방식

```
전송 O: "지출 - 커피 4500원 (5회), 주로 09시~10시"
전송 X: 사용자 이름, 위치, 원본 기록 데이터
```

## 대안

| 대안 | 제외 이유 |
|------|-----------|
| 온디바이스 TFLite 모델 | 학습 데이터 부족, 모델 크기 이슈, 정확도 낮음 |
| OpenAI GPT-4o | 성능 유사하나 Anthropic이 더 안전성·프라이버시 정책이 명확 |
| 단순 규칙 기반 (빈도 수) | 구현 쉽지만 복잡한 패턴 인식 불가, 차별화 없음 |
| Apple Intelligence / On-device LLM | iOS 18+ 한정, Android 미지원, API 불안정 |

## 결과

**긍정적:**
- 높은 제안 품질 (자연어 이해 기반)
- 클라이언트 앱에 ML 모델 번들 불필요 (앱 크기 절감)
- 모델 업그레이드 시 앱 업데이트 불필요

**부정적:**
- API 키 관리 필요 (사용자가 직접 발급)
- 인터넷 연결 없을 시 AI 제안 불가 (수동 입력으로 폴백 처리)
- API 사용 비용 발생 (사용량에 따라)
- 데이터가 Anthropic 서버를 통과함

## 폴백 처리

API 키 없음 / 네트워크 오류 / 패턴 데이터 부족(3회 미만) →
오류 없이 수동 입력 폼으로 자동 전환.
